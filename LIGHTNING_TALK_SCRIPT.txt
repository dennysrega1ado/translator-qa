## â˜ï¸ AWS Production Deployment (30 seconds - 10%)

"Finally, let's talk about **going to production on AWS EC2**. Five key steps:"

### 5-Step Deployment Process:

**1. Create EC2 Instance**
   - Launch `t3.medium` instance (2 vCPU, 4GB RAM)
   - Ubuntu 22.04 LTS
   - Attach Elastic IP (static public IP)
   - Configure Security Group: allow ports 22 (SSH) and 80 (HTTP)
   - **Cost: ~$30/month**

**2. Install Docker Environment**
   - SSH into instance
   - Install Docker and Docker Compose
   - Verify with `docker --version`

**3. Configure IAM Role for S3**
   - Create IAM role with S3 read/write permissions
   - Attach role to EC2 instance
   - No need for AWS credentials in code - uses instance profile
   - Secure and AWS best practice

**4. Set Environment Variables**
   - Create `.env` file with:
     - `DATABASE_URL` (PostgreSQL connection)
     - `AWS_S3_BUCKET` (your bucket name)
     - `SECRET_KEY` (generate secure 32+ char key)
     - `STORAGE_BACKEND=s3`
   - Update `docker-compose.yml` for production settings

**5. Deploy & Monitor**
   - Run `docker-compose up -d --build`
   - Access via `http://YOUR-ELASTIC-IP`
   - Set up automated backups with cron job
   - Monitor with `docker-compose logs -f`

### Cost Breakdown:
- EC2 t3.medium: $30/month
- EBS Storage 30GB: $2.40/month
- S3 Storage 50GB: $1.15/month
- **Total: ~$34/month**

"Full detailed instructions are in the `DEPLOYMENT_AWS_EC2.md` file in the repo."
